maven项目管理

maven简介

传统项目管理状态分析
  • jar包不统一，jar包不兼容
  • 工程升级维护过程操作繁琐
maven是什么？
  • Maven的本质是一个项目管理工具，将项目开发和管理过程抽象成一个项目对象模型（POM）
  • POM（Project Object Model）：项目对象模型

maven的作用
  • 项目构建：提供标准的，跨平台的自动化项目构建方式
  • 依赖管理：方便快捷的管理项目以来的资源（jar包），避免资源间的版本冲突问题
  • 统一开发结构：提供标准的，统一的项目结构

maven的安装与配置

maven环境变量配置
  • 依赖Java，需要配置JAVA_HOME
  • 设置maven自身的运行环境，需要配置MAVEN_HOME

maven基本概念

仓库

  • 仓库：用于存储资源，包含各种jar包

  • 仓库分类

    • 本地仓库：自己电脑上存储资源的maven仓库，连接远程仓库获取资源
    • 私服：公司或部门级的中央仓库。
    • 中央仓库:maven维护的统一仓库，存储所有资源
    • 私服的作用：
      • 保存具有版权的资源，包括购买或自主研发的jar（中央仓库的jar包都是开源的）
      • 一定范围内资源共享，仅对内部开放，不对外共享

坐标

  • 坐标：用于定位仓库中资源的位置
    maven坐标查询

  • Maven坐标主要组成吧

    • groupId： 定义当前Maven项目隶属 组织名称
    • artifactId： 定义当前Maven项目名称
    • version：定义当前版本号
    • socpe: 作用域
    • packaging：定义当前项目的打包方式
    <dependency>
    <groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>4.13.1</version>
    <scope>test</scope>
</dependency>

  • 作用：使用唯一标识，唯一性定位资源位置，通过该标识可以将资源的识别与下载交给机器完成

        </dependency>
    </dependencies>
</project>

依赖管理

依赖配置

  • 依赖指当前项目运行时需要的jar，一个项目可以由多个依赖

  • 格式：

    <!--所有依赖-->
<dependencies>
    <!--具体依赖-->
        <dependency>
            <!--组织Id-->
            <groupId>junit</groupId>
            <!--项目id>
            <artifactId>junit</artifactId>
            <!--依赖版本号
            <version>4.12</version>
        </dependency>
    </dependencies>
依赖传递
  • 依赖具有传递性
    • 直接依赖：当前项目中通过依赖配置建立起来的依赖关系
    • 间接依赖：被访问的资源如果依赖其他资源，当前项目间接依赖其他资源
依赖传递冲突问题
  • 路径优先：当资源中出现相同的资源时，层级越深，优先度越低，层级越浅，优先度越高。
  • 声明优先：当资源在相同层级被依赖时，配置顺序靠前的覆盖配置顺序靠后的
  • 特殊优先：当统计配置了相同资源的不同版本的，后配置的覆盖先配置的
依赖范围

​

  • 以来的jar默认情况下可以在任意地方使用，可以通过scope标签设定其作用范围

  • 作用范围：

    • 主程序范围有效(main文件夹范围内)
    • 测试范围有效（test文件夹范围内）
    • 是否参与打包（package指令范围内 ）

依赖范围传递性
  • 带有依赖范围的资源在进行传递时，作用范围将受到影响

模块聚合

多模块构建维护

  • 作用：聚合用于快速创建maven项目，一次性构建多个项目/模块。

  • 制作方式：

    • 创建一个空模块，在pom.xml中打包类型定义为pom

      <!--定义该工程用于进行构建管理-->
<packaging>pom</packaging>

    • 定义当前模块进行构建操作时关联的其他模块的名称

      <modules>
    <module>../ssm_controller</module>
    <module>../ssm_service</module>
    <module>../ssm_dao</module>
    <module>../ssm_pojo</module>
</modules>

  • 注意事项：参与聚合操作的模块最终执行顺序与模块之间的依赖关系有关，与配置顺序无关

模块继承

模块依赖关系维护

继承

  • 声明依赖管理:在父工程中声明进行模块管理

    <dependencyManagement>
    <!--具体的依赖-->
    <dependecies>
        <dependecy>
    	<groupId></groupId>
   		<artifactId></artifactId>
        <version></version>
    </dependecy>
    <dependecy>
    	<groupId></groupId>
   		<artifactId></artifactId>
        <version></version>
    </dependecy>
</dependecies>
</dependencyManagement>

  • 作用：通过继承可以实现在子工程中沿用父工程中的配置

    • Maven中的继承与java中的继承相似，在子工程中配置继承关系。
    • 子工程中可以省去组织Id与版本号

  • 制作方式：

    • 在子工程中声明其父工程坐标与对应的位置

      <!--定义该工程的父工程-->
<parent>
    <!--父工程的组织ID-->
	<groupId></groupId>
    <!--父工程的项目ID-->
    <artifactId></artifactId>
    <version></version>
    <!--父工程的pom文件-->
    <relativePath>路径/pom.xml</relativePath>
</parent>

  • 继承依赖使用：在子工程中定义依赖关系，无需声明依赖版本，版本参照父工程中依赖的版本

    <dependecies>
    <dependecy>
        <!--不用定义版本号-->
    	<groupId></groupId>
   		<artifactId></artifactId>
    </dependecy>
</dependecies>

继承的资源

继承与聚合
  • 作用：
    • 聚合用于快速构建项目
    • 继承用于快速配置
  • 相同点：
    • 聚合和继承的pom.xml文件打包方式均为pom，可以将两种关系制作在同一个pom文件中
    • 聚合和继承均属于设计型模块，并无实际的模块内容
  • 不同点：
    • 聚合是在当前模块中配置关系，聚合可以感知到参与聚合的模块有哪些
    • 继承是在子模块中配置关系，父模块无法感知到哪些子模块继承了自己
属性

属性类别：

自定义属性

  • 作用

    • 等同于定义变量，方便统一维护

  • 定义格式

    <properties>
	<junit.version>4.12</spring.version>
</properties>

  • 调用格式

    <dependency>
	<groupId>junit</groupId>
    <artifactId>junit</artifactId>
    <version>${junit.version}</version>
</dependency>
内置属性

  • 作用：

    • 调用Maven内置属性，快捷配置

  • 调用格式

    ${version}
${basedir}
Setting属性

  • 作用：

    • 使用Maven 配置文件setting.xml中的标签属性，用于动态配置

  • 调用格式

    ${settings.localRepository}
Java系统属性

  • 作用：

    • 读取Java系统属性

  • 调用格式

    ${user.home}

  • 系统属性查询方式

    mvn help:system
环境变量属性

  • 作用：

    • 调用环境变量属性

  • 调用格式

    ${env.JAVA_HOME}

资源配置多文件维护

配置文件引用pom属性

  • 作用

    • 在任意配置文件中加载pom文件中配置的属性

  • 调用格式

    ${jdbc.url}

  • 开启配置文件加载pom属性

    <resources>
	<resource>
        <!--设置支持配置文件对应的位置目录，支持使用属性动态设定路径-->
    	<directory>${project.basedir}/src/main/resources</directory>
        <filtering>true</filtering>
    </resource>
</resources>
多环境开发配置

  • 作用

    • 配置多环境开发需要的配置文件

  • 配置

    <!--创建多环境-->
<profiles>
    <!--具体环境-->
    <profile>
        <id><!--唯一id-->pro_env</id>
        <!--专属环境属性值-->
        <properties>
            <jdbc.url>jdbc::mysql://127.0.0.1:3306/dbname</jdbc.url>
        </properties>
        <!--设置默认启动-->
        <activation>
            <activeByDefault>true</activeByDefault>
        </activation>
    </profile>
    <profile>
        <id></id>
   		......
    </profile>
</profiles>
加载指定环境

  • 作用

    • 加载指定配置环境

  • 调用格式

    mvn 指令 -p 环境定义Id
mvn install -p pro_env
